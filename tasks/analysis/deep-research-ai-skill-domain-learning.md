## 1. 核心问题

当你接触一个全新领域（比如 Web3、合成生物学、量子计算），学习过程通常经历：

```
陌生 ──→ 了解 ──→ 框架 ──→ 重点 ──→ 追踪
 │        │        │        │        │
 │        │        │        │        └─ 持续跟进前沿动态
 │        │        │        └─ 找到20%的关键知识
 │        │        └─ 建立心智模型和知识地图
 │        └─ 知道这是什么、为什么重要
 └─ 一无所知
```

**核心问题：AI Skill 能否、以及如何系统性地加速这个过程中的每一步？**

## 2. 新领域学习的四个阶段

### 2.1 理论基础：三层学习模型

将 Bloom's Taxonomy（认知层级）和 DIKW Pyramid（信息层级）结合，形成领域学习的四阶段模型：

```
                    Bloom's Taxonomy
    ┌──────────────────────────────────────────────┐
    │ Create  ───── Stage 4: 追踪与创新            │
    │ Evaluate ──── Stage 3: 重点判断              │
    │ Analyze  ──── Stage 2: 框架建构              │
    │ Apply    ─────┘                              │
    │ Understand ─── Stage 1: 初步了解             │
    │ Remember  ─────┘                              │
    └──────────────────────────────────────────────┘
    
                    DIKW Pyramid
    ┌──────────────────────────────────────────────┐
    │ Wisdom  ────── Stage 4: 形成判断力           │
    │ Knowledge ──── Stage 3: 掌握关键知识         │
    │ Information ── Stage 2: 结构化理解           │
    │ Data ───────── Stage 1: 信息采集             │
    └──────────────────────────────────────────────┘
```

### 2.2 四阶段详细描述

#### Stage 1: 探索期（Explore）

```
状态: "我不知道我不知道什么"
目标: 建立直觉 — 这个领域是什么、解决什么问题、边界在哪
输入: 搜索引擎、维基百科、入门文章、播客
输出: 一段话能描述这个领域 + 10个核心术语
难点: 信息过载 vs 信息不足；分不清信号和噪声
```

#### Stage 2: 建构期（Structure）

```
状态: "我知道我不知道什么，但不知道怎么组织"
目标: 建立心智模型 — 领域的知识地图、关键概念之间的关系
输入: 教材目录、综述论文、领域专家的思维导图
输出: 概念关系图 + 分层知识框架
难点: 不同资料框架不一致；概念之间的连接比概念本身更重要
```

#### Stage 3: 聚焦期（Focus）

```
状态: "框架有了，但不知道哪些是关键"
目标: 找到 20% 的核心知识 — 领域的「第一性原理」和关键变量
输入: 经典论文、领域共识、关键争论点
输出: 核心概念清单 + 每个概念的「为什么重要」和「与什么相关」
难点: 深度需要时间；需要实践验证理解的正确性
```

#### Stage 4: 追踪期（Track）

```
状态: "我懂基础了，但领域在持续演进"
目标: 保持前沿 — 跟踪最新进展、辨别趋势信号
输入: 论文预印本、行业报告、社区讨论、关键人物动态
输出: 定期更新的领域动态摘要 + 对新信息的判断
难点: 信息持续涌入；需要判断什么是真正重要的
```

## 3. AI Skill 的能力地图

AI Skill 不是单一工具，而是一组可组合的能力。按学习阶段的需求，将 AI Skill 的能力分为六类：

### 3.1 能力分类

```
┌─────────────────────────────────────────────────────────────────┐
│                     AI Skill 能力金字塔                          │
│                                                                 │
│                        ┌─────────┐                              │
│                        │ 判断力  │ 评估信息质量、识别趋势       │
│                       ┌┴─────────┴┐                             │
│                       │  综合    │ 跨源整合、生成新洞察         │
│                      ┌┴───────────┴┐                            │
│                      │  分析    │ 拆解概念、对比观点            │
│                     ┌┴─────────────┴┘                            │
│                     │  组织    │ 结构化、分类、建立关联          │
│                    ┌┴───────────────┴┐                           │
│                    │  检索    │ 搜索、过滤、聚合信息             │
│                   ┌┴─────────────────┴┐                          │
│                   │  对话    │ 提问、解释、苏格拉底式引导        │
│                   └───────────────────┘                          │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 能力 × 工具映射

| 能力 | 当前 AI Skill 实现 | 代表工具/模式 |
|------|-------------------|---------------|
| **对话** | 通用 AI 助手的问答模式 | ChatGPT、Claude、DeepSeek |
| **检索** | Deep Research Agent、Web Search + RAG | Perplexity、Elicit、Scholar AI |
| **组织** | 知识图谱生成、自动标签、Zettelkasten 辅助 | Obsidian AI 插件、Notion AI |
| **分析** | 对比分析、概念拆解、矛盾点识别 | Claude Artifacts、ChatGPT Canvas |
| **综合** | 多源摘要、跨领域关联、框架生成 | Deep Research Report、Elicit |
| **判断** | 趋势追踪、信号检测、质量评估 | 定制 Agent + 监控脚本 |

### 3.3 关键洞察：能力的缺失层

当前 AI 工具在「检索」和「对话」层已经非常强，但在三个层面存在明显缺口：

1. **记忆与延续性** — AI 不记得你上周学到了什么，无法在新信息到来时与已有知识关联
2. **主动推送** — AI 只在被问到时才响应，无法主动告诉你「你关注的领域有新进展」
3. **元认知引导** — AI 可以回答问题，但不知道你「不知道什么」，无法主动填补盲区

**这正是 AI Skill 的设计空间：不是做更强的搜索，而是构建有记忆、有主动性的学习伙伴。**

## 4. Use Case 全景：6 个真实场景

### Use Case 1: 产品经理学习 Web3

```
人物: 张三，互联网产品经理，3年经验，零 Web3 基础
动机: 公司计划做 Web3 产品，需要在 2 周内建立基础认知
时间: 每天 1 小时，共 14 天
```

**Stage 1 · Day 1-3: 探索期**

| 步骤 | AI Skill 做什么 | 具体交互 |
|------|----------------|----------|
| 1. 一句话入门 | 生成一段 300 字的 Web3 核心概念描述 | "用产品经理能理解的语言解释 Web3" |
| 2. 术语表 | 列出 15 个必知术语，每个一句话解释 | "给我 Web3 领域最重要的15个术语" |
| 3. 类比映射 | 用已知概念映射未知概念 | "Web3 的 smart contract 类似传统产品的什么？" |
| 4. 边界识别 | 明确什么是 Web3、什么不是 | "Web3 和区块链是一回事吗？和加密货币呢？" |

**AI Skill 输出：**
```markdown
## Web3 核心概念速览
- 一句话: Web3 是用户拥有数据和资产所有权的下一代互联网
- 核心类比: Web1(读) → Web2(读+写) → Web3(读+写+拥有)
- 关键术语: 区块链, 智能合约, DeFi, NFT, DAO, Gas费, 钱包, DApp...
- 边界: Web3 ≠ 加密货币炒作; Web3 = 去中心化 + 用户主权 + 代币经济

## 你作为产品经理需要重点关注的
- 用户身份从「账号」变成「钱包地址」
- 产品治理从「中心化决策」变成「DAO 投票」
- 商业模式从「广告/订阅」变成「代币经济 + Gas」
```

**Stage 2 · Day 4-7: 建构期**

| 步骤 | AI Skill 做什么 | 具体交互 |
|------|----------------|----------|
| 1. 知识地图 | 生成 Web3 的概念关系图 | "画一个 Web3 的知识架构图" |
| 2. 学习路径 | 推荐最优学习顺序 | "我该先学 DeFi 还是先学 NFT？" |
| 3. 分层理解 | 基础/进阶/实战三层 | "Web3 的基础知识层、协议层、应用层各包含什么？" |
| 4. 盲区检测 | 主动发现并填补理解缺口 | "基于我目前的问题，我可能遗漏了哪些重要概念？" |

**AI Skill 输出（知识框架）：**
```
Web3 知识架构
├── 基础层
│   ├── 密码学基础（哈希、公私钥）
│   ├── 区块链原理（共识机制、出块）
│   └── 去中心化理念（为什么去中心化重要）
├── 协议层
│   ├── 以太坊（EVM、Gas、Solidity）
│   ├── Layer 2（Rollup、状态通道）
│   └── 跨链（桥、互操作性）
├── 应用层
│   ├── DeFi（借贷、DEX、稳定币）
│   ├── NFT / 数字资产
│   ├── DAO / 治理
│   └── SocialFi / GameFi
└── 体验层
    ├── 钱包设计（UX 挑战）
    ├── DApp 前端交互
    └── Onboarding 流程 ← 你作为 PM 最该关注
```

**Stage 3 · Day 8-11: 聚焦期**

| 步骤 | AI Skill 做什么 | 具体交互 |
|------|----------------|----------|
| 1. 第一性原理 | 提炼核心原理 | "Web3 的3个第一性原理是什么？" |
| 2. 争议追踪 | 列出领域内主要争论 | "Web3 领域最热的3个争议是什么？" |
| 3. 案例分析 | 深入1-2个经典案例 | "Uniswap 为什么成功？用 PM 视角分析" |
| 4. 实践任务 | 给出动手练习 | "帮我设计一个简单的 token 经济模型" |

**Stage 4 · Day 12-14+: 追踪期**

| 步骤 | AI Skill 做什么 | 具体交互 |
|------|----------------|----------|
| 1. 信息源推荐 | 推荐高质量信息源 | "哪些 Twitter 账号/Newsletter 值得关注？" |
| 2. 定期摘要 | 每周自动生成领域动态 | 配置 loop skill，每周一生成 Web3 周报 |
| 3. 信号检测 | 识别重大变化 | "这周有什么可能改变 Web3 格局的事件？" |
| 4. 学习闭环 | 检验理解是否准确 | "考我 5 个 Web3 概念，检查我的理解" |

### Use Case 2: 投资人研究合成生物学

```
人物: 李四，VC 合伙人，需要评估合成生物学赛道的投资机会
动机: 2 个 LP 推荐了合成生物项目，需要快速建立判断力
特点: 不需要成为专家，需要"问对问题"的能力
```

**AI Skill 差异化应用：**

| 阶段 | 投资人视角的 AI Skill | 和 Use Case 1 的差异 |
|------|----------------------|---------------------|
| 探索 | 生成「投资人必知的10个问题」而非概念 | 聚焦判断框架，不追求全懂 |
| 建构 | 画出产业链地图（上游原料→中游平台→下游应用） | 产业链视角而非知识体系 |
| 聚焦 | 分析市场规模、竞争格局、技术壁垒 | 投资决策需要的具体数据 |
| 追踪 | 监控融资事件、论文突破、政策变化 | 和投资决策直接相关的信号 |

**关键洞察：** 同一个领域，不同角色的人需要不同的 AI Skill 配置。产品经理关注用户和产品，投资人关注市场和壁垒，工程师关注技术细节。AI Skill 应该能根据角色自动调整。

### Use Case 3: 设计师学习 AI 绘图

```
人物: 王五，UI 设计师，想用 AI 提升设计效率
动机: 竞争压力 — 客户开始要求 AI 驱动的设计方案
特点: 强实践导向，学习速度 = 动手频率
```

**AI Skill 的特殊价值：** 这个场景中，「对话」能力的价值远大于「检索」— 设计师需要的是：
- 「我的设计稿和竞品比差在哪里？」→ AI 视觉分析
- 「帮我用 Midjourney 风格生成这个按钮的变体」→ AI 创作
- 「这个配色方案符合无障碍标准吗？」→ AI 评估

学习曲线不是线性的：设计师可以跳过理论直接上手，然后回过头补原理。AI Skill 应该支持「先做后学」的模式。

### Use Case 4: 退休教师学习用 AI 辅导孙辈

```
人物: 赵六，65岁退休数学教师，想用 AI 帮助孙子学习
动机: 孙子数学不好，自己想用「现代方法」帮忙
特点: 低数字素养、高信任需求、强情感驱动
```

**AI Skill 的特殊价值：** 这个场景的障碍不是信息获取，而是：
- 不知道 AI 能做什么 → 需要用生活场景举例
- 害怕操作复杂 → 需要语音交互，零技术门槛
- 担心 AI 给错误答案 → 需要可信来源标注

学习路径完全不同：不是从概念到应用，而是从「我能做什么」到「AI 还能帮我做什么」。

### Use Case 5: 自由职业者学习个人品牌建设

```
人物: 陈七，自由翻译，想建立个人品牌获客
动机: AI 冲击翻译市场，需要差异化竞争
特点: 学以致用，学习的产出必须是可见的行动
```

**AI Skill 的特殊价值：** 学习和行动不可分离：
- 学到「内容营销」→ AI 立刻帮写第一篇 LinkedIn 帖子
- 学到「定位理论」→ AI 立刻帮分析自己的差异化优势
- 学到「SEO」→ AI 立刻帮优化个人网站

每个学习点都必须有一个可执行的输出。AI Skill 是「学习+执行」一体化的。

### Use Case 6: 企业安全团队学习零信任架构

```
人物: 周八，企业 IT 安全负责人，需要评估零信任架构迁移方案
动机: 公司被要求在 6 个月内完成零信任改造
特点: 高风险决策、需要权威信息、有合规要求
```

**AI Skill 的特殊价值：** 这个场景的关键是「可信赖」：
- 不能用 AI 生成的内容直接作为方案 → 需要 AI 标注每个信息的来源
- 不能遗漏合规要求 → AI 需要覆盖 NIST、ISO 27001 等标准
- 需要向领导汇报 → AI 需要生成不同层级的摘要（给 CTO 的 vs 给工程师的）

### 5.3 每个 Stage 的理想 AI Skill 配置

#### Stage 1 配置（探索期）

```
触发: 用户输入"我想了解 [领域名]"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Skill 1: Domain Overview Generator
  - 输入: 领域名 + 用户角色
  - 输出: 300字概述 + 15个核心术语 + 3个类比
  
Skill 2: Landscape Scanner  
  - 输入: 领域名
  - 输出: 主要玩家/流派/争议的概览
  
Skill 3: Learning Path Recommender
  - 输入: 领域名 + 可用时间 + 用户角色
  - 输出: 个性化学习路径（含时间估算）
```

#### Stage 2 配置（建构期）

```
触发: 用户说"帮我整理一下 [领域] 的知识框架"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Skill 4: Knowledge Graph Builder
  - 输入: 用户已学的概念列表
  - 输出: 概念关系图（Mermaid/ASCII）
  
Skill 5: Concept Connector
  - 输入: 两个概念
  - 输出: 它们的关系 + 为什么这种关系重要
  
Skill 6: Blind Spot Detector
  - 输入: 用户目前的理解描述
  - 输出: 可能遗漏的关键概念 + 填补建议
```

#### Stage 3 配置（聚焦期）

```
触发: 用户说"深入讲讲 [具体概念]" 或 "这个领域的核心是什么"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Skill 7: First Principles Extractor
  - 输入: 领域名
  - 输出: 3-5个第一性原理 + 每个的解释
  
Skill 8: Debate Tracker
  - 输入: 领域名
  - 输出: 前3个争议 + 各方观点 + 核心分歧点
  
Skill 9: Case Study Analyzer
  - 输入: 具体案例名
  - 输出: 成功/失败因素 + 关键决策点 + 可迁移洞察
  
Skill 10: Practice Generator
  - 输入: 概念 + 用户水平
  - 输出: 动手练习题 + 验证标准
```

#### Stage 4 配置（追踪期）

```
触发: 定时触发（loop skill）或用户说"最近 [领域] 有什么新进展"
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Skill 11: Signal Detector
  - 输入: 领域名 + 关注维度
  - 输出: 过去7天的重大变化 + 影响评估
  
Skill 12: Weekly Digest Generator
  - 输入: 订阅的信息源
  - 输出: 本周领域动态摘要（分重要/一般/参考）
  
Skill 13: Knowledge Refresh
  - 输入: 用户的知识库
  - 输出: 需要更新的概念 + 新概念 + 过时概念
  
Skill 14: Trend Forecaster
  - 输入: 近3个月领域动态
  - 输出: 趋势判断 + 概率评估 + 验证指标
```

+ 主动性   ████   █████  ████   ████   ████   █████

### 6.2 三个关键差异

**差异 1: 有记忆的学习**

```
传统 AI:
  用户: "Web3 的 Gas 费是什么？"
  AI: [解释 Gas 费]
  # 下次对话完全不记得用户已经学过这个

AI Skill:
  用户: "Web3 的 Gas 费是什么？"
  Skill: [解释 Gas 费] → 记录到学习档案
  # 下次：
  Skill: "你之前学了 Gas 费，现在有个相关概念叫 
         'EIP-1559 Gas 机制改革'，要不要了解？"
```

**差异 2: 知道你不知道什么**

```
传统 AI:
  用户问什么就答什么，不知道用户的知识盲区

AI Skill:
  Skill: "你已经理解了 DeFi 的借贷协议，
         但你还没接触过 '清算机制' — 这是 
         DeFi 风险管理的核心概念。"
  # 主动填补盲区，基于用户已学内容推断未学内容
```

**差异 3: 从被动到主动**

```
传统 AI:
  等用户提问 → 回答 → 等下一个问题

AI Skill:
  主动推送: "你关注的 Uniswap 刚发布了 V4 白皮书，
            其中有一个新概念叫 'Hooks'，
            和你之前学的智能合约直接相关。"
  主动检验: "上个月你学了 Web3 基础，来做个小测验？"
  主动连接: "你学的 DAO 治理概念，
            和你之前了解的公司治理框架有关联，看看？"
```

### 6.3 AI Skill 与传统工具的本质区别

| 维度 | 传统 AI 助手 | AI Skill 学习系统 |
|------|-------------|------------------|
| **交互模式** | 一问一答 | 引导式学习流 |
| **记忆** | 无（或窗口内短期记忆） | 持久学习档案 |
| **主动性** | 被动响应 | 主动推荐/检验/更新 |
| **个性化** | 基于当前对话 | 基于完整学习历史 |
| **输出** | 答案 | 学习成果（概念图/笔记/测验） |
| **时间维度** | 即时 | 跨天/跨周/跨月 |
| **角色感知** | 无 | 根据用户角色调整深度和视角 |
| **闭环** | 无 | 学→练→测→更新 |

## 8. 对产品设计的启示

### 8.1 AI Skill 学习系统的设计原则

**原则 1: 从输出倒推输入**
不要问「AI 能教什么」，而是问「用户学完需要产出什么」。不同角色的产出不同：
- PM → 产品方案文档
- 投资人 → 投资备忘录
- 设计师 → 设计作品集
- 教师 → 教案/辅导计划

**原则 2: 学习路径不是线性的**
不要强制「Stage 1 → 2 → 3 → 4」的顺序。支持：
- **跳级**: 设计师可以直接从 Stage 3（动手做）开始
- **回退**: 投资人看完案例后可能需要回退到 Stage 1 补基础
- **并行**: 同时处于不同阶段的多个领域

**原则 3: 可组合的 Skill 架构**
每个学习功能是一个独立的 Skill，可以自由组合：
```
Domain Overview Skill + Investment Lens Skill + Signal Detector Skill
= 完整的 VC 领域学习工具

Domain Overview Skill + Practice Generator Skill + Knowledge Refresh Skill  
= 完整的工程师技能学习工具
```

**原则 4: 渐进式披露**
不要一次给太多信息。每个交互只给用户「下一步刚好需要的」：
- Day 1: 一段话 + 10 个术语
- Day 3: 一个框架图 + 学习路径
- Day 7: 3 个深入话题 + 练习
- Day 14+: 定期更新 + 新发现

### 8.2 技术实现的关键组件

│              AI Skill 学习系统架构                     │

### 8.3 冷启动策略

新用户第一天怎么开始？三个关键设计：

1. **角色选择器**: 「你是 PM / 工程师 / 设计师 / 投资人 / 学生 / 其他？」
   → 决定学习视角和深度

2. **领域输入器**: 「你想学什么？输入一个领域名或上传一篇文章」
   → 触发 Stage 1 的 Domain Overview Skill

3. **时间承诺**: 「你每天能投入多少时间？」
   → 决定学习路径的节奏
