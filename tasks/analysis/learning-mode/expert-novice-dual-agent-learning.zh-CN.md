# Expert/Novice 双代理学习模式 — 可行性分析

## 1. 概念概述

### 1.1 核心思路

在群聊中创建两个 AI 代理，通过协作来教授人类学习者：

| 角色 | 职责 | 行为模式 |
|------|------|---------|
| **Expert（专家）** | 解释概念，引导学习路径 | 使用费曼技巧 / 苏格拉底式提问 |
| **Novice（新手）** | 跟随 Expert 学习，提出问题 | 展示理解程度，发现知识缺口 |

人类学习者通过观察或参与 Expert-Novice 对话来学习：
1. Expert 的结构化讲解
2. Novice 提出的问题（学习者可能也有同样的问题）
3. Expert 对 Novice 问题的回答
4. Novice 尝试应用知识的过程

### 1.2 学习模型

```
┌─────────────────────────────────────────────────┐
│                  群聊空间                         │
│                                                   │
│   Expert ──讲解概念──▶ Novice                     │
│     │                     │                       │
│     │   回答问题           │  提出疑问              │
│     │   ◀─────────────────│                       │
│     │                     │                       │
│     └──────┬──────────────┘                       │
│            │                                      │
│            ▼                                      │
│     人类学习者（观察 / 参与）                        │
└─────────────────────────────────────────────────┘
```

## 2. 可行性分析

### 2.1 技术可行性：✅ 可行

| 方面 | 评估 | 说明 |
|------|------|------|
| 多代理聊天 | ✅ 成熟 | OpenClaw 支持 skills + souls；LangChain/AutoGen/CrewAI 均支持多代理 |
| 角色化提示 | ✅ 已验证 | 系统提示可以可靠地定义 Expert/Novice 人格 |
| 上下文共享 | ✅ 可用 | 共享聊天历史提供共同上下文 |
| 记忆持久化 | ⚠️ 需要设计 | 需要跟踪跨会话的学习进度 |
| 技能组合 | ✅ 可用 | 技能可以叠加（如费曼 + 苏格拉底） |

### 2.2 学习效果：✅ 前景良好

基于研究的优势：
- **费曼技巧**：简单解释概念能暴露知识缺口（Expert 为 Novice 做这件事）
- **苏格拉底式提问**：引导式提问激活批判性思维（Expert 向 Novice 提问）
- **门生效应**：教别人能强化自身学习（Novice "反向教学"）
- **社会性学习**：观察对话比阅读文档更有吸引力

### 2.3 主要风险

| 风险 | 缓解措施 |
|------|---------|
| Expert 幻觉产生错误解释 | 添加验证技能；交叉引用文档 |
| Novice 提出琐碎问题 | 可配置深度等级；上下文感知提问 |
| 对话循环无进展 | 进度跟踪；学习目标设定 |
| 上下文窗口溢出 | 摘要技能；分块学习 |

## 3. 输入 / 输出设计

### 3.1 课程类型 A：agents.md 教程（如"如何使用这个仓库"）

#### 输入

```
agents.md  (教程文档)
├── 前置条件
├── 安装步骤
├── 使用示例
├── 配置选项
└── 故障排除
```

**Expert 输入：**
- 完整的 agents.md 内容作为知识库
- 课程学习目标（从文档结构自动生成）
- 学习者假设水平（初学者 / 中级）

**Novice 输入：**
- 学习者背景（如"我了解 git 基础，但从未使用过 agents"）
- 学习目标（如"我想理解如何创建自己的 agent"）
- 当前理解状态（通过 memory 跟踪）

#### 输出

```
学习会话输出：
├── 结构化对话记录
├── 识别的关键概念
├── 问答对（Novice 的问题 + Expert 的回答）
├── 生成的练习题
├── 知识检查点（类似测验的验证）
└── 进度总结
```

#### 对话流程示例

```
Expert: 今天我们来学习如何使用 agents.md。首先，agents.md 定义了
        AI 代理在你的项目中的行为方式。把它想象成你的 AI 助手的
        "工作描述"。你觉得一个代理需要知道什么才能做好它的工
        作？

Novice: 嗯……也许它需要知道有哪些工具可用？还有项目结构是
        什么样的？

Expert: 完全正确！在 agents.md 中，我们定义：
        1. 可用工具（脚本、命令）
        2. 项目结构感知
        3. 行为规则（做什么 / 不做什么）
        你能给我一个你想让代理遵循的"行为规则"的例子吗？

Novice: 也许……"不要在没有询问的情况下删除文件"？

Expert: 完美的例子！这正是你放在 agents.md 中的安全规则。
        现在让我给你展示实际的格式……
```

### 3.2 课程类型 B：通用课程（如 references 文件夹中的材料）

#### 输入

```
课程材料
├── 讲义 / 教材章节
├── 代码示例
├── 图表 / 架构描述
├── 练习题
└── 评估标准
```

**Expert 输入：**
- 课程内容作为知识库
- 课程结构（章节、前置条件、依赖关系）
- 教学策略（费曼优先、苏格拉底优先或混合）

**Novice 输入：**
- 学习者先验知识评估
- 学习节奏偏好
- 偏好学习方式（代码示例 vs 类比 vs 视觉）

#### 输出

```
学习会话输出：
├── 概念讲解（由 Expert 简化）
├── 类比库（Expert 生成的类比）
├── 问答日志（Novice 的问题 + Expert 的回答）
├── 带注释的代码示例
├── 练习题及解答
├── 自测测验
├── 学习进度报告
└── 下次会话建议
```

## 4. 实现设计

### 4.1 所需技能

| 技能 | 角色 | 用途 | 来源 |
|------|------|------|------|
| **Feynman Explainer** | Expert | 将复杂概念拆解为简单解释 | 自定义技能 |
| **Socratic Questioner** | Expert | 引导式提问以检验理解 | 自定义技能 |
| **Active Learner** | Novice | 提出澄清性问题，请求示例/代码 | 自定义技能 |
| **Knowledge Validator** | Expert | 在继续前进前验证 Novice 的理解 | 自定义技能 |
| **Progress Tracker** | 系统 | 跟踪学习进度，生成报告 | 自定义技能 |
| **Content Parser** | 系统 | 将课程材料解析为可教学块 | 自定义技能 |

### 4.2 所需 Souls

```yaml
# Expert Soul
name: learning-expert
description: |
  经验丰富的教师，擅长使用费曼技巧拆解复杂话题。
  耐心、鼓励、精通苏格拉底式提问。
  能根据学习者水平调整解释深度。
traits:
  - 耐心
  - 有条理
  - 鼓励性
  - 精确
rules:
  - 始终在推进前验证理解
  - 使用学习者领域的类比
  - 不跳过基础概念
  - 教授技术话题时提供代码示例
  - 庆祝正确回答，温和纠正错误

---
# Novice Soul
name: learning-novice
description: |
  充满好奇心的学习者，通过提出有深度的问题来建立理解。
  不害怕说"我不明白"，总是请求具体示例。
  代表指定知识水平的学习者。
traits:
  - 好奇
  - 对知识缺口坦诚
  - 坚持
  - 务实
rules:
  - 当概念抽象时总是请求示例
  - 对技术话题请求代码演示
  - 当解释过于浅显时指出
  - 在请求更多帮助前尝试应用概念
  - 用自己的话总结理解
```

### 4.3 所需记忆

```
记忆架构：
```

## 5. 技能设计详情

### 5.1 Feynman Explainer 技能

```yaml
---
name: feynman-explainer
description: |
  使用费曼技巧拆解复杂技术概念。
  简化为核心思想，使用类比，发现缺口。
type: teaching
---

## 指令

解释概念时：

1. **简单陈述概念** — 一句话，无术语
2. **向初学者解释** — 像教一个 12 岁的孩子
3. **发现缺口** — 哪里难以简化？那就是缺口所在
4. **使用类比** — 连接日常经验
5. **回顾和精炼** — 能否更简单？

### 技术内容要求：
- 始终提供代码示例
- 明确展示输入 → 输出
- 先解释"为什么"，再解释"怎么做"

### 反模式：
- ❌ 使用未定义的术语
- ❌ 跳过"显而易见"的步骤
- ❌ 假设课程中未涵盖的先验知识
- ❌ 解释"是什么"而不解释"为什么"
```

### 5.2 Active Learner 技能

```yaml
---
name: active-learner
description: |
  通过提出澄清性问题、请求示例、展示理解来主动学习。
type: learning
---

## 指令

收到解释后：

1. **复述** — 用自己的话重述以验证理解
2. **提问** — 对不清楚的地方发问：
   - "你能给我一个真实的例子吗？"
   - "如果 [边缘情况] 会怎样？"
   - "这与 [之前的概念] 有什么关系？"
3. **应用** — 尝试在新场景中使用概念
4. **发现缺口** — 说明还有什么不理解

### 问题类型：
- **澄清型**："在这个上下文中 X 是什么意思？"
- **示例型**："你能给我展示代码示例吗？"
- **关联型**："这与我们之前学的有什么关系？"
- **挑战型**："如果我们用 Y 而不是 X 会怎样？"
- **应用型**："所以如果我想做 Z，我应该……？"
```

## 6. OpenClaw 特定实现

### 6.1 OpenClaw 架构

```
OpenClaw 学习系统：
├── Skills（通过 clawhub 安装）
│   ├── feynman-explainer/
│   │   └── SKILL.md
│   ├── active-learner/
│   │   └── SKILL.md
│   ├── content-parser/
│   │   ├── SKILL.md
│   │   └── scripts/parse-course.py
│   └── progress-tracker/
│       ├── SKILL.md
│       └── scripts/track-progress.py
├── Souls（通过 onlycrabs.ai 注册）
│   ├── learning-expert/SOUL.md
│   └── learning-novice/SOUL.md
└── Memory（本地持久化）
    ├── course-state.json
    ├── expert-memory.json
    └── novice-memory.json
```

### 6.2 代理配置

```yaml
# Expert 代理
agent:
  name: Learning Expert
  soul: learning-expert
  skills:
    - feynman-explainer
    - socratic-questioner
    - knowledge-validator
  memory: persistent
  temperature: 0.3  # 更事实性，更少创造性

# Novice 代理
agent:
  name: Learning Novice
  soul: learning-novice
  skills:
    - active-learner
  memory: persistent
  temperature: 0.7  # 更多样化的问题
```

### 6.3 会话流程

```
1. 用户提供课程材料（URL / 文件路径 / 文本）
2. Content Parser 技能将材料分块
3. 自动生成学习目标
4. 会话开始：
   a. Expert 引入第一个概念（费曼风格）
   b. Novice 复述并提出问题
   c. Expert 回答并验证理解
   d. Progress Tracker 更新状态
   e. 对下一个概念重复
5. 会话结束：
   a. 已学概念总结
   b. 练习题
   c. 自测测验
   d. 下次会话建议
```

## 7. 评估标准

| 指标 | 目标 | 度量方式 |
|------|------|---------|
| 概念覆盖率 | > 90% 的课程材料 | 主题清单 |
| 理解验证 | > 80% 检查点通过 | 知识检查点 |
| 问题质量 | > 70% 非琐碎问题 | 手动 / LLM 评估 |
| 会话参与度 | 每个概念 > 5 轮对话 | 对话长度 |
| 学习保持率 | 24 小时后 > 70% | 随访测验 |

## 8. 结论

### 可行性：✅ 技术可行，教育前景良好

**为什么可行：**
- Expert-Novice 对话天然实现主动学习
- 费曼技巧 + 苏格拉底式提问是经过验证的教学组合
- 多代理 AI 框架（OpenClaw、CrewAI、AutoGen）提供基础设施
- 记忆系统支持跨学习会话的连续性

**成功关键：**
1. 精心设计的 souls（人格）用于 Expert 和 Novice
2. 结构化的技能设计（费曼 + 苏格拉底 + 主动学习）
3. 持久化记忆用于跟踪学习进度
4. Content Parser 能合理地将材料分块

**建议下一步：**
1. 用单个 agents.md 教程做原型
2. 测试 Expert soul 的费曼技巧提示
3. 测试 Novice soul 的主动学习提示
4. 用真实学习者评估对话质量
5. 根据反馈迭代技能
